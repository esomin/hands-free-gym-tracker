# 시뮬레이터 — 트래커 앱 연결 사용법

## 서비스 구성

```
시뮬레이터 (Streamlit :8501)
    │  WebSocket 센서 데이터 전송
    ▼
백엔드 (FastAPI :8000)
    │  파이프라인 처리 후 이벤트 브로드캐스트
    ▼
트래커 앱 (React :5173)
```

---

## 1단계 — 서비스 시작

프로젝트 루트에서 한 번에 실행합니다.

```bash
./dev.sh
```

| 서비스 | 주소 |
|---|---|
| 트래커 앱 | http://localhost:5173 |
| 시뮬레이터 | http://localhost:8501 |
| 백엔드 API | http://localhost:8000/docs |

---

## 2단계 — 기구 미등록 시나리오 (모달 팝업 확인)

> 처음 실행하는 경우 기구가 하나도 등록되지 않은 상태입니다.
> 아래 순서를 따르면 **기구 등록 모달**이 나타납니다.

### 순서

**① 트래커 앱 열기** — http://localhost:5173
- 헤더에 "연결됨" 배지가 표시되면 백엔드 연결 성공

**② 시뮬레이터 열기** — http://localhost:8501

**③ 파라미터 설정**
- **기구 선택**: 원하는 기구 선택 (예: 레그프레스)
- **텀블러 상태**: `🚶 이동 중` 선택 (기본값)
- **노이즈 레벨**: `0.5` 이상 권장 (이동 중 신호가 백엔드 임계값을 넘어야 함)

**④ 전송 시작** — `▶ 전송 시작` 버튼 클릭
- 시뮬레이터 우측에 "🟢 전송 중" 표시 확인
- 트래커 앱에 "이동 중" 배지가 나타남

**⑤ 거치됨으로 전환** — `📍 거치됨 (기구 점유 중)` 버튼 클릭
- 약 0.5초 후 트래커 앱에 "거치됨" 배지로 변경
- **기구 등록 모달 팝업** — 미등록 기구이므로 등록 요청

**⑥ 기구 이름 입력 후 등록**
- 예: "레그프레스" 입력 → 등록 버튼 클릭
- 모달이 닫힘

---

## 3단계 — 기구 재인식 시나리오 (등록 후 확인)

기구가 등록된 이후 동일한 기구를 다시 가져다 놓으면 `equipment_detected` 이벤트가 발생합니다.

**① 이동 중으로 전환** (시뮬레이터에서 `🚶 이동 중` 클릭)
**② 다시 거치됨으로 전환** (`📍 거치됨` 클릭)
**③** 트래커 앱에 기구명과 신뢰도가 표시됨

---

## 주의사항

| 항목 | 설명 |
|---|---|
| 노이즈 레벨 | **0.5 이상** 설정 권장. 너무 낮으면(0.1 이하) 이동 중 신호가 약해 백엔드가 이미 거치됨으로 판정할 수 있음 |
| 전이 순서 | 반드시 `이동 중 → 전송 시작 → 거치됨` 순서로 진행. 이미 거치됨 상태에서 시작하면 전이 감지 불가 |
| 서버 재시작 | 백엔드 재시작 시 세션 캐시와 지문 저장소가 초기화됨. 등록한 기구 정보가 사라지므로 재등록 필요 |
| MongoDB | `docker compose up -d` 로 MongoDB가 실행 중이어야 스마트 디폴트 API가 동작함 |

## 개발 중 빠른 테스트 (dev 트리거)

시뮬레이터 없이 curl로 이벤트를 직접 발생시킬 수 있습니다.

```bash
# 미등록 기구 감지 이벤트 → 등록 모달 팝업
curl -X POST http://localhost:8000/dev/trigger/user-1/equipment-unknown

# 텀블러 거치됨 이벤트 → 배지 변경
curl -X POST "http://localhost:8000/dev/trigger/user-1/tumbler-state?state=settled"

# 기구 인식 이벤트 → SmartDefault 카드 표시
curl -X POST http://localhost:8000/dev/trigger/user-1/equipment-detected
```
