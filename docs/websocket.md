# WebSocket 아키텍처


## 연결 구조

```
┌─────────────┐        ┌───────────────────────────┐        ┌─────────────┐
│  시뮬레이터    │──연결──▶│                           │◀──연결──│  브라우저     │
│             │  센서   │   /ws/user-1              │  이벤트  │  (프론트엔드) │
│             │──데이터─▶│   (백엔드 WebSocket 채널)    │──전송──▶│            │
└─────────────┘        └───────────────────────────┘        └─────────────┘
```

- **시뮬레이터**: 센서 데이터를 백엔드로 **보내는** 역할
- **브라우저**: 백엔드에서 가공된 이벤트를 **받는** 역할
- **백엔드**: 센서 데이터를 처리하고, 상태 변화가 생기면 연결된 모든 클라이언트에게 이벤트를 **브로드캐스트**

---

## 동작 순서

```
① 시뮬레이터  →  /ws/user-1 연결
② 브라우저    →  /ws/user-1 연결
   (이 시점에 /ws/user-1 채널에 클라이언트 2개가 연결된 상태)

③ 시뮬레이터가 센서 데이터 전송
   ┌─────────────────────────────────────────┐
   │ {                                       │
   │   "accel_magnitude": 1.28,  ← 가속도      │
   │   "gyro_magnitude":  0.91,  ← 자이로      │
   │   "timestamp": "2026-03-11T10:00:00Z"   │
   │ }                                       │
   └─────────────────────────────────────────┘

④ 백엔드 처리
   - SensorReading 생성
   - 슬라이딩 윈도우(deque)에 추가
   - detect_tumbler_state() 호출
   - 상태 변화 감지: "이동 중" → "거치됨"

⑤ 상태가 바뀐 경우에만 브로드캐스트
   ┌──────────────────────────────────────────────────────────┐
   │ {                                                        │
   │   "type": "tumbler_state_changed",                       │
   │   "payload": {                                           │
   │     "state": "settled",                                  │
   │     "transitioned_at": "2026-03-11T10:00:00.123456Z"     │
   │   },                                                     │
   │   "timestamp": "2026-03-11T10:00:00.123456Z"             │
   │ }                                                        │
   └──────────────────────────────────────────────────────────┘
   → 시뮬레이터에게도 전송 (무시해도 됨)
   → 브라우저에게도 전송 → 화면 업데이트
```
