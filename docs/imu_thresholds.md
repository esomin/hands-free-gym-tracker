`backend/pipeline/imu_state.py` 의 상수(`WINDOW_SIZE`, `MOVE_ACCEL_THRESHOLD`, `MOVE_GYRO_VAR`) 결정 과정을 기록합니다.

## 텀블러 상태별 센서 기준값

### 가속도 (accel_magnitude, 단위: g)

텀블러가 정지해 있을 때 가속도계는 지구 중력만 측정하므로 **1.0 g** 에 수렴합니다.
이동 중에는 진동·충격이 더해져 1.0 g 에서 벗어납니다.

| 상태 | accel_magnitude | 중력 기준 편차 \|accel − 1.0 g\| |
|------|----------------|----------------------------------|
| 거치됨 (순수) | 1.000 ± 0.015 g | ≈ 0.000 ~ 0.020 g |
| 이동 중 (순수) | 1.0 ± 0.35 g | ≈ 0.200 ~ 0.350 g |

> 순수 구간: 슬라이딩 윈도우가 이전 상태 샘플을 모두 교체한 이후를 기준으로 측정.

### 자이로 (gyro_magnitude, 단위: rad/s)

텀블러가 정지해 있으면 회전이 없으므로 자이로 값이 **0 에 수렴**합니다.
이동 중에는 손목·팔의 회전에 따라 자이로 값이 활성화됩니다.

| 상태 | gyro_magnitude | 슬라이딩 윈도우 분산 (gyro_var) |
|------|---------------|--------------------------------|
| 거치됨 (순수) | ≈ 0.000 ~ 0.015 rad/s | ≈ 0.00002 (rad/s)² |
| 이동 중 (순수) | ≈ 0.6 ~ 2.4 rad/s | ≈ 0.356 (rad/s)² |

> **주의**: 각 구간 시작 직후 WINDOW_SIZE 개(= 0.4 s) 샘플은 이전 상태의 값이 윈도우에 혼재하는 전환 지연 구간입니다.
> 세그먼트 전체 평균에는 이 구간이 포함되어 거치됨 gyro_var 가 0.04~0.06 으로 높게 나타납니다.
> 실제 순수 거치됨 gyro_var 는 0.00002 이며, 세그먼트 평균값은 참고용으로만 사용해야 합니다.

## 임계값 결정

### WINDOW_SIZE = 20

50 Hz 샘플레이트 기준으로 **0.4 초** 구간을 커버합니다.
단발 노이즈 억제와 상태 전이 반응 속도 사이의 균형점으로 선택했습니다.

- 너무 작으면(예: 5): 노이즈 1개에 상태가 바뀌는 불안정
- 너무 크면(예: 100 = 2 s): 전환 감지가 느려 실사용성 저하

### MOVE_ACCEL_THRESHOLD = 0.04 g

`notebooks/imu_state_dev.ipynb` 그리드 탐색 결과, 정확도를 최대화한 값입니다.

```
순수 거치됨 편차 ≈ 0.000 ~ 0.020 g   →   0.04 보다 충분히 아래
순수 이동 중 편차 ≈ 0.200 ~ 0.350 g  →   0.04 보다 충분히 위
```

두 분포 사이의 간격이 매우 넓어, 0.04 ~ 0.12 범위 내 어떤 값도 유사한 결과를 냅니다.
그리드 탐색에서 0.04 가 전환 지연 구간의 오탐을 가장 적게 발생시켰습니다.

### MOVE_GYRO_VAR = 0.40 (rad/s)²

직관과 달리 **순수 이동 중 gyro_var(≈ 0.356) 보다 높은 값**입니다.
아래 탐색 결과가 이 설정의 근거입니다.

```
THRESH_GYRO=0.05  →  gyro 단독 발동 8샘플 중 오탐 6, 정답 2  →  정확도 0.9383
THRESH_GYRO=0.20  →  gyro 단독 발동 4샘플 중 오탐 4, 정답 0  →  정확도 0.9383
THRESH_GYRO=0.40  →  gyro 단독 발동 0샘플                    →  정확도 0.9450 ← 채택
```

**gyro 조건이 단독으로 발동하는 샘플은 항상 오탐입니다.**
발생 원인은 이동 → 거치됨 전환 직후의 지연 구간입니다.
이 시점에 accel_dev 는 이미 0.04 아래로 떨어졌지만,
윈도우에 이동 샘플이 남아 있어 gyro_var 는 아직 높습니다.
임계값을 낮출수록 이 구간에서 gyro 조건이 더 오래 발동하여 거치됨을 이동으로 잘못 판단합니다.

결론적으로 **accel 조건(0.04 g)이 이동 감지를 전담**하고,
gyro 조건(0.40)은 accel 이 반응하지 못하는 극단적 회전에 대비한 보조 역할로만 남습니다.

## 판별 규칙 요약

```python
# OR 조건: 하나라도 초과하면 이동 중
if accel_dev_mean > 0.04 or gyro_var > 0.40:
    state = 'moving'
else:
    state = 'settled'
```

| 조건 | 역할 |
|------|------|
| `accel_dev_mean > 0.04` | 이동 감지 주력 |
| `gyro_var > 0.40` | 극단적 회전에 대한 보조 안전장치 |
